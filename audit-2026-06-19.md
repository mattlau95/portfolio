# UX Audit Report — Matthew C. Lau Portfolio

> **Date:** 2026-06-19 · **Reviewed:** `index.html`, `projects/edison-dental.html` (source), `http://localhost:4321` (Part A) · **Tools run:** Lighthouse 12, axe-core 4.11.4, pa11y, manual code + keyboard review

---

## Executive summary

A clean, well-structured static portfolio with strong accessibility fundamentals — semantic HTML, skip link, `prefers-reduced-motion` respected throughout, and both axe-core and pa11y report zero violations. The main issues are a WCAG 2.2 touch-target size failure on all sidebar nav and social links, and the `h1` name being sized too small relative to the section headings. Speed Index runs high locally (6.6s) due to async Google Fonts swapping, but LCP and CLS are both within threshold. **1 blocker, 3 important, 4 polish.**

## 🔴 P0 — Blockers

| # | Finding | Checklist ref | Evidence | Fix | Effort |
|---|---------|---------------|----------|-----|--------|
| 1 | Nav and social links fail WCAG 2.2 touch target size (SC 2.5.8) — all 5 links flagged | §8 — Target Size (WCAG 2.2) | Lighthouse `target-size` audit: `nav > ul > li > a` (×3) and `.socials > li > a` (×2) all below the 24×24 px minimum or lack 24px spacing clearance from adjacent targets | Add `padding-inline: var(--space-3)` to `nav a` and `.socials a` in `style.css`. This widens the clickable zone and ensures the 24px circle clearance requirement is met. | ⚡ |

## 🟡 P1 — Important

| # | Finding | Checklist ref | Evidence | Fix | Effort |
|---|---------|---------------|----------|-----|--------|
| 1 | `h1` name sized at `--text-lg` (max 1.25rem / 20px) — smaller than section `h2` headings (`--text-2xl`, max 2.75rem) | §10 — Visual hierarchy | `style.css:154–159` — `.identity-name { font-size: var(--text-lg) }`. The name, as the dominant identity element, should outsize body text and feel like a hero. User-confirmed issue. | Bump `.identity-name` to `--text-xl` (max 1.875rem) or `--text-2xl` (max 2.75rem). `--text-xl` is the safer choice to avoid overflow in the 25%-wide sidebar. | ⚡ |
| 2 | Speed Index 6.6s — content appears visually stable late due to async Google Fonts swap | §9 — Performance | Lighthouse Speed Index 0.38 score. Fonts load via `onload` trick; fallback text renders first, then swaps when web fonts arrive — Speed Index captures that delayed visual stabilization. LCP (1.4s) and CLS (0) are both fine. | Add `<link rel="preload" as="style">` for the Google Fonts CSS URL, or switch to `font-display: optional` to avoid FOUT entirely (text stays with fallback if fonts don't load within the FCP window). Note: `optional` trades FOUT for occasional fallback rendering on slow connections. | 🔨 |
| 3 | No nav item is initially active on page load | §1 — System status | `scripts/main.js:36` calls `setActive('about')` but no section carries `id="about"`. The three sections are `#projects`, `#experience`, `#process`. On load, zero nav links are highlighted until the user scrolls into a tracked section. | Change `setActive('about')` → `setActive('projects')` since Projects is the first visible section and the page renders scrolled to top. | ⚡ |

## What passed

- **Semantic HTML throughout** — proper `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, `<time>`, heading hierarchy (`h1` → `h2` → `h3`)
- **Accessible focus states** — `:focus-visible` with 2px `--color-accent` outline, skip link functional
- **`prefers-reduced-motion` respected** — spotlight, transitions, scroll-behavior, and group-hover all conditioned
- **All meaningful images have alt text** — profile photo (primary), project thumbnails, decorative secondary photo is `aria-hidden="true"`
- **Color contrast** — `--color-text: #e8e8e2` on `--color-bg: #0f0f0e` is well above AA; muted text (#a8a8a2) clears ~8.5:1
- **CLS = 0** — explicit `width`/`height` on all images and a fixed-size profile photo container prevent layout shift
- **LCP 1.4s** — within threshold; async fonts loaded with preconnect + noscript fallback
- **axe-core + pa11y: zero violations**
- **No forms, no destructive actions, no AI output, no user data** — §2–6 are N/A and the site is correspondingly clean
- **Nav scroll-spy** — IntersectionObserver implementation is smooth and respects the viewport correctly
- **External link security** — all `target="_blank"` links carry `rel="noopener noreferrer"` ✓
