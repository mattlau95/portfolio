## What passed

- **Semantic HTML** — real `<nav>`/`<main>`/`<section>`/headings/`<a>` throughout, no div soup (only two grouping `<div>`s, both legitimate for pure flex layout).
- **Keyboard operability** — every interactive element is a real `<a>`, reachable and operable with Tab/Enter by default; tab order matches visual/DOM order.
- **Focus Not Obscured (WCAG 2.2)** — exhaustively checked, not just spot-checked: all 6 focusable links inside `<main>` on `index.html`, tested at 5 scroll positions (30 element/position combinations), **zero** instances of the sticky sidebar overlapping a focused element.
- **Visible focus states** — global `:focus-visible` outline present, never stripped.
- **Color contrast** — independently computed via the WCAG relative-luminance formula (not just eyeballed): muted text **8.03:1**, body text **15.59:1**, accent **10.06:1** against the `#0f0f0e` background — all comfortably clear of the 4.5:1 AA floor (muted/accent even clear AAA's 7:1).
- **Meaning not conveyed by color alone** — the active nav state changes both color *and* indicator-line width; the featured project card uses a border *and* an explicit "★ Featured case study" label, not just color.
- **Target Size — nav links and résumé link** — nav links measure 29–31px tall (clear of the 24px floor); résumé link 24–26px (passes, though under the 44px comfort aim). `.badges`/`.tags` pills are correctly non-interactive (no `<a>`/`<button>` inside), so WCAG 2.5.8 doesn't apply to them — confirmed, not a violation.
- **Responsive** — no horizontal scroll or cut-off content at 375px; the 1024px sticky-to-stacked breakpoint collapse works cleanly.
- **Reduced-motion / touch guards on the spotlight** — both a JS removal (touch devices, `prefers-reduced-motion`) and a CSS `display:none` fallback.
- **Core Web Vitals** — CLS excellent on both pages (0.013 / 0.042, both well under 0.1); TBT (INP proxy) is 0ms on both pages; index.html's LCP (2.2s) already clears the 2.5s threshold.
- **pa11y** — zero issues on both pages.
- **axe-core** — zero violations on the case study page; the single violation on `index.html` is the `<h1>` issue already listed above.
- **No render-blocking JS** — `main.js` loads with `defer`.
- **Font loading** — `font-display: swap` already in place via the Google Fonts URL.
- **External links** — all `target="_blank"` links correctly pair with `rel="noopener noreferrer"`.
- **Sections 1–6 (system feedback, destructive actions, errors, forms, AI UX, data loss)** — not applicable to this site: no forms, no async actions, no destructive operations, no AI-generated content surfaced to visitors, no user data entry to lose. Correctly N/A, not silently skipped.
- **Images/alt text** — no `<img>` tags exist yet (thumbnails are CSS placeholders pending real assets); alt-text requirements correctly deferred rather than retrofitted onto placeholders.
