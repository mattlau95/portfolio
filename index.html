How I Build

Every project I ship starts from a written spec and ends with a formal audit — built with Claude Code, tracked in Linear, with Figma in the loop when the UI calls for it.

  • /_ capture

    Capture, then triage

    Quick notes go into an inbox via a custom /capture command; a separate /triage pass turns them into scoped Linear issues.

  • system.md

    Spec before code

    Every project starts as a version-controlled build plan, keeping the architecture and the implementation in perfect sync. This site was built that way.

  • ↳ linear

    Continuous momentum

    One isolated project per application, running on weekly cycles. Every repository maintains an active devlog to document progress and micro-decisions.

  • /_ audit

    UX as tooling

    Design quality and WCAG 2.2 accessibility compliance shouldn't rely on memory. They live inside a repeatable, programmatic /audit command.

  • figma + mcp

    Design when the UI demands it

    Complex UI gets designed in Figma first — token-based variables that map directly to the codebase. Then Figma MCP hands off to Claude Code, which takes over implementation and keeps design and code in sync without manual translation.
